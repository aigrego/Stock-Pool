# 系统架构文档

## 概述

股票池管理系统采用分离式架构，将数据收集和股票管理解耦，通过 WebSocket 实时通信。

## 组件说明

### 1. 数据收集器 (Data Collector)

**位置**: kimiclaw 服务器

**职责**:
- 定时从多个数据源抓取实时行情
- 按优先级自动故障转移
- 通过 WebSocket 推送给股票管理系统
- 维护本地股票池缓存

**配置**:
```bash
DATA_SOURCES=sina,tencent,eastmoney  # 数据源优先级
FETCH_INTERVAL=5                     # 抓取间隔(秒)
WS_TARGET_URL=ws://vercel-app.com    # WebSocket 目标
```

**部署**:
```bash
cd data-collector
npm install
npm run build
pm2 start ecosystem.config.js
```

### 2. 股票管理系统 (Stock Manager)

**开发环境**: kimiclaw:3000
**生产环境**: Vercel

**职责**:
- 股票池的 CRUD 管理
- 接收数据收集器的 WebSocket 推送
- 广播实时行情给所有前端客户端
- K线图和技术指标计算
- 预警规则检查和飞书推送

**API 端点**:
```
GET    /api/stocks          # 获取所有股票
POST   /api/stocks          # 创建股票
PUT    /api/stocks/:code    # 更新股票
DELETE /api/stocks/:code    # 删除股票
GET    /api/stats           # 统计数据
GET    /api/realtime        # 实时行情(轮询备份)
GET    /api/alerts/check    # 检查预警
GET    /api/socket          # Socket.io 端点
```

### 3. 前端 (Frontend)

**技术栈**: Next.js + React + TypeScript + Tailwind CSS + shadcn/ui

**实时数据流**:
1. 数据收集器抓取行情
2. 推送到 Vercel Socket.io 服务端
3. 服务端广播给所有连接的前端
4. 前端实时更新 UI

## 部署配置

### kimiclaw (数据收集器)

```bash
# 安装依赖
cd /root/workspaces/feishu-groups/shared-stockpool/data-collector
npm install

# 编译
npm run build

# PM2 启动
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### Vercel (股票管理系统)

```bash
# 部署
cd /root/workspaces/feishu-groups/shared-stockpool
vercel --prod
```

**环境变量**:
```
DATABASE_URL=mysql://user:password@host:port/stockpool?ssl=true
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
```

## 优势

1. **解耦**: 数据收集和股票管理独立，互不影响
2. **实时性**: WebSocket 推送，延迟 < 1秒
3. **扩展性**: 可轻松添加更多数据源或前端客户端
4. **容错性**: 多数据源自动故障转移
5. **成本**: 利用 Vercel Hobby 免费额度，只需维护 kimiclaw 服务

## 注意事项

1. **CORS**: Vercel Socket.io 需要配置跨域
2. **连接稳定性**: WebSocket 需要处理重连逻辑
3. **数据一致性**: 多个前端客户端看到的数据一致
4. **安全性**: 数据收集器需验证身份，防止非法推送
